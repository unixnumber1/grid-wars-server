# Grid Wars — UI/UX Аудит

Полный анализ всех экранов, попапов, кнопок, меню и взаимодействий в `public/index.html` (~22K строк).

---

## CRITICAL — влияет на геймплей или вызывает потерю ресурсов

### 1. Двойные клики на покупках без debounce
**Где:** кнопки покупок в магазине, прокачки предметов, инвестирования скиллов
- `buyMythic()` (строка ~14361) — нет disable кнопки ДО try-блока, можно купить дважды
- `buyStars()` (строка ~14231) — onclick без throttle
- `buyCorePack()` (строка ~14575) — onclick без debounce
- `investInSkill()` (строка ~9578) — inline onclick, нет блокировки кнопки
- `buyInventorySlot()` (строка ~7078) — нет state lock

**Импакт:** Игрок может потерять алмазы/осколки/эфир из-за двойного тапа.

**Фикс:** Добавить `btn.disabled = true` в начало каждого async-обработчика + `finally { btn.disabled = false }`.

### 2. Размещение HQ без подтверждения
**Где:** `btn-place-hq` (строка 10714)
- Нет confirmation dialog
- Кнопка не блокируется на время API-запроса
- Случайный тап = штаб поставлен навсегда

**Фикс:** Добавить `confirm()` или кастомный диалог перед размещением.

### 3. Silent catch на критичных операциях
**Где:** 
- `deactivatePin()` строка 7856: `.catch(() => {})` — fail при отвязке пина = позиция рассинхронизирована
- `doTeleportToHq()` строка 7878: `.catch(() => {})` — fail при телепорте = сервер не знает позицию
- Несколько мест в admin-панели

**Импакт:** Игрок думает, что действие выполнено, но сервер не обработал его.

**Фикс:** Показывать toast при ошибке, откатывать UI-состояние.

### 4. Race condition в боевой системе (touch + mouse)
**Где:** строка 17399-17404
- `touchstart` + `mousedown` на одной кнопке — на некоторых устройствах оба срабатывают
- Может привести к двойному удару за один тап

**Фикс:** Добавить guard `if (_atkHolding) return;` в `startHold()`.

## HIGH — заметно ухудшает UX

### 5. Нет loading state при размещении HQ
**Где:** строка 10714-10742
- Кнопка остаётся активной во время API-запроса
- Нет spinner/индикатора загрузки

### 6. Нет loading state при открытии маркета
**Где:** строка ~16287
- 1300мс timeout перед показом попапа после flyTo
- Жёсткий timeout вместо callback на завершение анимации
- Если сеть медленная, UI "зависает"

### 7. SetInterval утечки памяти
**Где:**
- `_vaseTimerInterval` (строка ~11837)
- `_vaseCleanupInterval` (строка ~11845)  
- Таймер HP-регена шахт (строка ~7537)
- Floating mine timer (строка ~6657)

**Импакт:** Интервалы накапливаются при переключении экранов, утечка памяти, фантомные обновления.

**Фикс:** Очищать все интервалы при закрытии попапов/смене экрана.

### 8. Socket listeners без cleanup
**Где:** `initSocket()` (строки 11000+) — 59+ слушателей
- Никогда не удаляются при дисконнекте
- При реконнекте дублируются
- Нет валидации входящих данных

### 9. Кнопки admin-панели зависают при таймауте
**Где:** строки 12289-12383
- `btn-fix-positions`, `btn-fix-usernames`, `btn-spawn-vases` и т.д.
- Кнопка disabled = true, но если apiFetch зависнет — навсегда заблокирована
- Нет finally-блока для восстановления

### 10. Нет подтверждения при смене имени (стоит 50 алмазов)
**Где:** строка ~13386-13410
- Стоимость показана в модалке, но нет финального confirm
- Случайный тап = минус 50 алмазов

## MEDIUM — неудобства, не баги

### 11. Маленькие touch target'ы (< 48px)
**Где:**
- `.marker-emoji` — 32×32px (строка ~422) — маркеры на карте
- `.level-option` — 36×36px (строка ~560) — выбор уровня в настройках  
- `.speed-dot` — 10×10px (строка ~1911) — переключение скорости джойстика
- `.walk-ms-btn` — padding 3px 10px (строка ~1070) — кнопки наград за ходьбу

**Apple/Google рекомендуют минимум 44-48px.**

### 12. Нет hover-состояний на desktop
**Где:** практически все кнопки имеют только `:active`, но не `:hover`:
- `.burger-item`, `.settings-tab`, `.map-style-card`, `.avatar-cell`, `.lb-refresh`
- `.profile-badge-card`, `.btn-profile` и другие

**Импакт:** На desktop (Telegram Desktop) кнопки не дают визуальный feedback при наведении.

### 13. Попапы: несогласованное поведение закрытия
- `popup-overlay` — закрывается по тапу на фон (строка ~8557)
- `change-username-overlay` — НЕ закрывается по тапу на фон (stopPropagation, строка ~3250)
- `avatar-sheet-overlay` — НЕ закрывается по тапу на фон (строка ~3472)
- `box-odds-popup` — нет backdrop click handler

**Ожидание пользователя:** тап за пределами = закрыть. Нарушение во многих местах.

### 14. Нет empty state в инвентаре ядер
**Где:** строка ~1624
- Показывает пустые ячейки с прочерками вместо осмысленного сообщения
- Нет подсказки "где взять ядра"

### 15. Непоследовательные сообщения об ошибках
- Одни функции: `❌ ${err.message}` 
- Другие: `❌ ${msg}`
- Третьи: `t('toast.collect_error', ...)` через i18n
- Часть ошибок кулдауна фильтруется, часть нет

### 16. Хардкод задержки вместо animation callbacks
**Где:** строки ~10111-10114, ~10134-10137, ~16316-16322
- `setTimeout(300)` вместо `transitionend`
- `setTimeout(1300)` вместо callback на `map.flyTo()`

**Импакт:** На медленных устройствах — рассинхрон UI и анимации.

### 17. Toast с max-width 280px
**Где:** строка ~407
- На маленьких экранах текст обрезается
- Длинные сообщения об ошибках не помещаются

### 18. Грид предметов/ядер не адаптивный
**Где:** строки ~1610, ~1614
- `grid-template-columns: repeat(4, 1fr)` без media queries
- На очень маленьких экранах (< 320px) ячейки слишком мелкие
- На больших экранах (планшеты) — слишком крупные

### 19. Кнопка крафта фиксирована внизу без safe-area
**Где:** строка ~1852
- `position: fixed; bottom: 0` — может перекрываться клавиатурой или safe area
- На iPhone с home indicator — кнопка перекрыта

### 20. Нет Enter-key для отправки форм
**Где:**
- Ввод username (строка ~13362) — нет обработчика Enter
- Ввод координат для телепорта (строка ~3371) — нет Enter handler
- Поиск игроков в admin (строка ~1959) — нет Enter handler

## LOW — мелочи и улучшения

### 21. Accessibility
- Кнопки закрытия используют эмодзи ✕ вместо SVG/иконки
- Нет `aria-label` на кнопках с иконками
- Нет `aria-modal="true"` на модальных окнах
- Нет `role="dialog"` на попапах
- Ввод имени: placeholder вместо `<label>`

### 22. Admin дублирование display:none
**Где:** строка ~3462
- `display:none;display:none;` — CSS ошибка (не влияет, но грязный код)

### 23. RequestAnimationFrame без cleanup
**Где:** строка ~7743
- `requestAnimationFrame` переменная переиспользуется без `cancelAnimationFrame`
- При быстром зуме/драге накапливаются вызовы

### 24. Spinner 32px — слишком маленький
**Где:** строка ~463
- Loading spinner всего 32×32px
- На больших экранах едва заметен

### 25. Попап HP бар — владельцы построек
**Где:** строка ~827
- `.player-shield-dome` с `inset: -4px` может вылезать за пределы родителя
- На узких экранах — визуальный глитч
